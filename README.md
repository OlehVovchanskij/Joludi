# Система аналізу телеметрії та 3D-візуалізації польотів БПЛА

Веб-рішення для автоматичного розбору Ardupilot логів, обчислення кінематичних метрик місії, побудови 2D/3D траєкторії та формування AI-висновку.

## Що реалізовано

- Обробка телеметрії з бінарних логів Ardupilot (`.bin`): виділення GPS та IMU повідомлень.
- Розрахунок частот дискретизації та одиниць вимірювання.
- Ядро аналітики:
  - загальна дистанція через формулу haversine;
  - швидкість з IMU через трапецієвидне інтегрування;
  - максимальна горизонтальна/вертикальна швидкість;
  - максимальне прискорення;
  - максимальний набір висоти;
  - тривалість польоту.
- Конвертація WGS-84 -> ENU та побудова 3D траєкторії.
- Інтерактивний дашборд:
  - завантаження файлу через UI;
  - OSM карта з анімованим промальовуванням треку;
  - 3D ENU візуалізація;
  - панель метрик та sampling rates.
- AI-асистент:
  - endpoint для генерації текстового висновку;
  - LLM-режим (за наявності API ключа);
  - fallback rule-based висновок без зовнішнього API.

## Архітектура

- `backend/` - FastAPI + `pymavlink`, `pandas`, `numpy`, `scipy`, `plotly`.
- `frontend/` - Next.js + Tailwind + Leaflet (OSM) + Plotly + Framer Motion.

## Чому цей стек

- FastAPI швидко обробляє multipart-upload і повертає JSON для аналітики.
- `pymavlink` напряму працює з форматами польотних контролерів Ardupilot.
- `numpy/pandas/scipy` зручні для обчислень і нормалізації сирих телеметричних даних.
- Plotly дає інтерактивну 3D сцену ENU без написання власного рендерера.
- Next.js дозволяє швидко зібрати web dashboard для операторів/аналітиків.

## Запуск через Docker (обов'язковий для здачі)

### 1. Запуск всього рішення

```bash
docker compose up --build
```

Сервіси:
- Frontend: http://localhost:3000
- Backend: http://localhost:8501

### 2. Змінні середовища (опційно)

- `NEXT_PUBLIC_API_BASE` - URL бекенду для фронта.
- `GOOGLE_MAPS_API_KEY` - ключ для Google static map payload (опційно).
- `OPENAI_API_KEY` - ключ для LLM AI summary (опційно).
- `OPENAI_BASE_URL` - сумісний OpenAI endpoint.
- `OPENAI_MODEL` - назва моделі.

## Локальний запуск без Docker

### Backend

```bash
cd backend
pip install -r requirements.txt
uvicorn app:app --host 0.0.0.0 --port 8501 --reload
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

## Математичні примітки

- Haversine використовується для обчислення довжини сегментів траєкторії між GPS-точками на сферичній моделі Землі.
- Перетворення WGS-84 -> ENU виконується відносно стартової точки для локальної метричної системи координат.
- Трапецієвидне інтегрування IMU застосовується для оцінки швидкості з дискретного профілю прискорення.
- При подвійному інтегруванні IMU можливий суттєвий дрейф; у виробничих системах це компенсується фільтрами з GPS/баро/візуальними даними.
